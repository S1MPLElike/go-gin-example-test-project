## Bug总结
### 问题原因
在 tag_service/tag.go 的 GetAll() 方法中，构建 cache_service.Tag 对象时 漏掉了 Name 参数 ：

### 导致的问题
- 所有不同 name 参数的请求都生成 相同的缓存键
- 第一次请求后，后续所有请求都命中同一个缓存
- 表现为：不管传什么 name 值，返回的都是第一次请求的数据
### 修复方案
添加 Name: t.Name 字段：

```
// 修复后
cache := cache_service.Tag{
    Name:   t.Name,      // ✅ 已添加
    State:  t.State,
    PageNum:  t.PageNum,
    PageSize: t.PageSize,
}
```
